# Week 2026-W24 (Jun 8–12)

Two clear blocks: a security hardening sprint (Mon–Wed) and building an SEO engine end to
end (Thu–Fri), with its documentation.

## Backend (fintech SaaS platform, client)
- **Shipped (security):** secrets out of version control with correct ignore rules; an
  SSRF-safe scraping wrapper (Decorator pattern, fail-closed on DNS); stopped leaking
  exception detail in debug endpoints; a reusable dependency (Proxy pattern) closing IDOR
  on resource-keyed endpoints; identity derived exclusively from the verified JWT; a
  billing upgrade path that provisions seats; and the removal of ~900 lines of dead code.
- **Shipped (SEO engine):** 7 tables + ORM + Alembic migration; full implementation with
  Search Console sync, blog writing, ideas, and WordPress publishing plus Celery tasks
  (95 tests); async blog generation over Celery + SSE; a blog detail endpoint with
  auth/ORM hardening; and a CORS fix for the tenant header.
- **Technical decisions:** Decorator for SSRF; centralized authorization in a dependency;
  identity only from the verified session; async + SSE for long jobs; partial indexes for
  opportunity queries.
- **Mistakes (root cause):** wrong tenant key on the SEO engine (organization → user →
  finer-grained, two migrations) — root cause: I didn't check how auth resolves identity
  before designing the schema. One migration also shipped a commit late.
- **What I learned:** "measure before deleting" (instrument, confirm zero traffic) turns
  dead-code removal into evidence-backed security work.
- **Where I'm weak:** data-architecture decisions at the start of a feature — I tend to
  start coding the schema before fixing the tenant key.

## Documentation
- **Shipped:** full SEO-engine documentation (~25 files): frontend routes/api-client,
  backend endpoints, the async + streaming flow, the editor spec, and the tenant
  migration.
- **What I learned:** documenting the same day I build keeps the spec faithful to the real
  implementation, not the intent.

## Retrospective
- **Pattern of the week:** high output, but skipping the up-front data-architecture
  decision was paid back in rework (three tenant keys, one late migration).
- **Habit to change:** write a three-line ADR before touching a new feature's schema, and
  always ship model + migration + services in the same commit.
- **Logged debt:** rotate potentially exposed credentials still in git history.
