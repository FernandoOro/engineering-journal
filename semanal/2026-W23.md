# Week 2026-W23 (Jun 4–5)

A short week (joined mid-week). Focus: consistency of a collaborative chat flow and an
admin coupon-activation feature, with integration and deployment.

## Backend (fintech SaaS platform, client)
- **Shipped:** fixed the collaborative chat so owner and members work on a single shared
  document and return correct conflict codes per question, removing a guard that blocked
  the owner; built the admin upgrade-coupon endpoints (47 tests). Integrated and deployed
  to `main`.
- **Technical decisions:** one chat = one document (remap members to the owner on
  read/write); enforce answer uniqueness at the question level, not the package level,
  because the answers table lacks per-user attribution.
- **Mistakes (root cause):** the first conflict guard blocked the owner — root cause: the
  data model can't distinguish "my pending answers" from "someone else's committed ones".
- **What I learned:** collaborative-consistency bugs are usually data-model bugs in
  disguise.
- **Where I'm weak:** I tend to patch symptoms before centralizing the rule; the
  "effective owner" logic ended up split across modules.

## Retrospective
- **Pattern of the week:** patching symptoms (conflict guards) before addressing the root
  cause (a per-user model for a shared document).
- **Habit to change:** for any consistency bug, sketch the data model before touching the
  logic.
