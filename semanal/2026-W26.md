# Week 2026-W26 (Mon Jun 22 – Fri Jun 26)

A week of **hardening and parity**: consolidating a third-party integration's authentication into a
single manager (killing the static tokens and closing a standing security finding), making the
**Dockerized dev environment behave like production** (CI secrets, DB schema, credentials, task
queues), and stabilizing an **internal multi-agent product** (from crashing to parallel). The thread
running through the whole week was **dev↔prod parity**: making what runs in dev genuinely match what
runs in prod. Foundation work, with a lot of **testing** and **deploy-with-worker-restart** at every
step.

## Backend (fintech SaaS platform — client)

- **Done (quality/security — OAuth consolidation):** a single OAuth manager replacing **3 OAuth
  implementations** and **6 duplicated refresh routines**. ADR + pilot with ~110 lines of tests,
  service-by-service migration **6/6** via Adapters, an `is_surface_configured()` method, and a
  static-token cleanup. **Removed the long-lived `*_ACCESS_TOKEN` env vars** → closes a standing
  finding by design. Hotfix: a connection test now gates on `is_surface_configured`.
- **Done (webhook security):** a robust inbound webhook — token via header **or** body + a payload
  Adapter — plus a **periodic OAuth-surface health-check + alert** against silent failure, a footprint
  audit, a per-surface liveness smoke-test, and a re-auth runbook.
- **Done (dev Dockerization / CI):** granular `deploy-dev` CI + Dockerized Postgres + a dev compose
  aligned to local; **schema bootstrap** (revert a misguided migration → `create_all`+reconcile → a
  portable `schema_baseline.sql` dump); **credential parity**: use the prod service account for the
  managed-ML service in dev — fixing an autocomplete bug; worker alignment to prod (specific research
  concurrency, no maintenance worker). API keys → independent CI secrets.
- **Done (critical bug — non-terminal `DONE` state):** an infinite re-execution loop because `DONE`
  wasn't terminal. Fixed with a **3-layer defense + cross-worker Redis lock** + ~146 lines of tests: a
  guard in the canonical writer, an execution gate, a `terminal_lock` (15-min TTL), neutralizing a
  cascade invalidation, and a recovery carve-out for the resurrection healer.
- **Done (queue homogenization):** a `maintenance` queue sat **orphaned** in dev (prod has no
  maintenance worker) → the periodic layer never ran. Collapsed `maintenance` → `default`; validated
  the healer runs and `ALMOST_DONE → DONE` progresses.
- **Done (multi-agent product):** a crash fix (`AttributeError` in agent resolution), **per-agent /
  per-model latency observability** on dispatch and the swarm path, and performance: **parallelize the
  swarm's sub-phase**, an **SSE idle-heartbeat + total cap**, and a **per-agent safety cap**.
- **Done (connector):** an ad-platform REST API v20 → v21 bump in the connector + its agent.
- **Technical decisions:**
  - **One authentication point** (manager + an Adapter per surface), never 3 copies.
  - **Killing the static token** closes the risk class by design, not by patch.
  - **Bootstrap, not migration**, for tables the model creates; better still, a portable baseline dump.
  - **dev↔prod parity includes credentials**: the binary was identical, the service account wasn't.
  - **A terminal state defended at every layer** + a Redis lock for the worker race.
  - **A queue with no consumer = work that falls through the floor**; collapse onto `default`.
  - **Instrument before optimizing**: per-agent latency pointed at the serial sub-phase.
  - **SSE needs a heartbeat** on long jobs; a **per-agent cap** so none hangs the swarm.
- **Mistakes (root cause):**
  - The OAuth consolidation and a "seats" piece were **cut in half by an AI credit limit** because I
    didn't slice them into small, self-contained PRs from the start.
  - Burned a cycle adding+reverting a migration by not distinguishing "table the model creates" from
    "schema change to migrate".
  - The connection-test hotfix was predictable: I removed an attribute without grepping all its
    readers.
  - The infinite loop existed because `DONE` was **never terminal by design**.
- **What I learned:** a client-side "timeout" is often a **crash in disguise** (the flow dies, it
  doesn't lag); confirm the flow isn't dying before optimizing. And a state machine should declare its
  forbidden transitions up front, not get patched with four guards after the fact.
- **Where I'm weak:** slicing big refactors so a credit limit doesn't split them; **automated
  regression** coverage for multi-agent systems (today I test every prompt by hand); and startup
  checks that catch dev↔prod divergence on their own (a queue with no consumer, a missing credential).
- **Next-week focus:** finish the "seats" piece; capture the multi-agent prompts as a **regression
  set** (smoke); resolve why the image **isn't pushing to the registry** + close the account upgrade
  for image **scanning**; re-baseline the migration chain.

## Operations / DevOps (cross-cutting, every day)

days:

  terminal-state fix and the queue realignment), each with its own validation; deploying the Docker
  image to dev through the week; and **two simultaneous deploys to main + production** monitored on
  Tuesday.
- **Secrets in CI:** migrated each API key to an **independent secret** and non-sensitive config to
  *Variables*; end-to-end validation that the container **reads them from CI** and not a leftover
  `.env`; the registry-PAT + server `docker login` + CI-credentials flow.
- **Server administration:** direct work on dev/prod; granting **production admin access** to a
  teammate (Fri).
- **Per-environment `.env` / credentials:** the prod service account for the managed-ML service in
  dev, the webhook token var, folding the `.env` residue into the compose; syncing across environments
  so one change doesn't break another.
- **DB schema and data:** full-schema bootstrap in dev (`upgrade`+`create_all`+reconcile), a portable
  baseline dump of the real schema; a clean start tested every round.
- **Container lifecycle + testing:** build/up/restart of the stack with healthcheck verification;
  testing takes its time while workers restart and come back healthy — especially the **exhaustive
  testing across every prompt** for the multi-agent product (the bulk of Friday).
- **Recurring git/delivery:** dev↔main merges, integrating a teammate's features (email-domain

## Global retrospective
- **Pattern of the week:** everything orbited **dev↔prod parity** — credentials, DB schema, task
  queues, workers, secrets. Half the bugs (autocomplete, the orphaned queue, the re-execution loop)
  were **environment divergences**, not logic bugs. Homogenizing is what buys the confidence to
  deploy.
- **Habit to change:** **slice big refactors** into small PRs so a credit limit doesn't leave them
  half-done; and build **startup checks** that catch dev↔prod divergence automatically ("every
  declared queue has a consumer", "every required credential is present").
- **Logged debt (accumulated):** the unfinished "seats" piece; image not pushing to the registry +
  scanning; migration-chain re-baseline; a regression set for the multi-agent product; a worker-queue
  selection that omits some queues.
