## Learnings / technical decisions
- **Telemetry pays for itself: it made a silent failure visible.** Instrumenting usage wasn't only for
  budgets — persisting `model/provider` **revealed** that the managed LLM had been down for a while
  and everything was going through fallback. What you don't measure, you don't see: the silent
  fallover only surfaced because we now **record which model actually served**.
- **Direct `await` > fire-and-forget for writes that matter.** Persisting the run "in the background"
  risked losing it if the request finished first; if the data matters, wait for it.
- **A typed error has a shape contract.** The 429 had to be a plain string; returning a nested object
  breaks whoever consumes the error. The spec's acceptance contract includes the **shape** of the
  error body.
- **Every provider ships its own `usage` shape.** The fallback provider nests it differently; assuming
  one shape leaves tokens null. Boundary capture has to **normalize per provider**.
- **Done with real evidence, not "it compiles".** The method forced me to paste production integration
  evidence before marking Done — and that testing is exactly what caught the credentials bug.

## What I could do better
- The silent fallover could have been caught **earlier**: a fallback that fires **every time** should
  **alert**, not just be logged. There's a missing "the primary never serves" alarm on top of the log.
- I split the boundary/telemetry work across two days again due to the token limit. The week's lesson
  is clear: **slice into small PRs from the spec** (which the method already asks) and start the big
  work early.
